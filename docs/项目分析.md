**步骤 9：立即显示 UI + 后台网络请求**

```cpp
    QSettings settings;
    QString lastCityCode = settings.value("lastCityCode").toString();
    if (!lastCityCode.isEmpty() && m_cache->load(lastCityCode, mToday, mDay))
        updataUI();
    else
        showDefaultUI();

    QTimer::singleShot(0, this, [this, lastCityCode]() {
        if (!lastCityCode.isEmpty())
            m_apiClient->getWeatherInfo(lastCityCode);
        else
            m_apiClient->getLocationByIP();
    });
```

- **零等待原则**：先调用 UI 函数立即渲染（缓存数据或占位符），用户感知不到加载延迟。
- **缓存命中** → `m_cache->load` 填入真实数据，`updataUI()` 展示。
- **无缓存** → `showDefaultUI()` 直接设置占位符（"--℃"、"加载中..."、"--"），不依赖 `weatherdata.h` 的默认构造值。
- **QTimer::singleShot(0)** 将网络请求推迟到事件循环启动后执行，避免 `m_highChart`/`m_lowChart` 尚未初始化就被访问。
- 三级回退：缓存命中 → 显示缓存 + 后台刷新；有历史无缓存 → 显示占位符 + 请求历史城市；首次 → 显示占位符 + IP 定位。

### 2.3 缓存 + 网络请求决策流程

Widget 构造函数中，三级回退判断：

```
Widget 构造函数
  +-- QSettings 读取 lastCityCode
  |     +-- 为空（首次运行）
  |     |     +-- showDefaultUI() → 占位符
  |     |     +-- ApiClient::getLocationByIP()
  |     |           +-- GET https://ipinfo.io/json
  |     |           +-- 解析 city 拼音
  |     |           +-- getCityCodeByPinyin()
  |     |           +-- getWeatherInfo(cityCode)
  |     +-- 非空
  |           +-- DataCache::load(lastCityCode)
  |           |     +-- 命中且未过期 → updataUI()（跳过网络）
  |           |     +-- 未命中或过期 → showDefaultUI()
  |           |           +-- ApiClient::getWeatherInfo(lastCityCode)
  |           |                 +-- 判断格式：已是编码 or 城市名查表
  |           |                 +-- 构造 URL + GET 请求
  +-- 等待数据（来自缓存或网络）
```

### 2.4 数据返回后的信号链与 UI 更新

```
服务器返回数据
  +-- QNetworkAccessManager::finished(QNetworkReply*)
        +-- ApiClient::onReplied()
              +-- status != 200 或 reply->error()
              |     +-- m_retryCount == 0 → emit errorOccurred（弹窗提示用户）
              |     +-- ++m_retryCount <= MAX_RETRY_COUNT (10)
              |     |     +-- m_retryTimer->start(30000ms) → retry()
              |     |           +-- 重新 GET 请求（后台静默重试）
              |     +-- m_retryCount > MAX_RETRY_COUNT
              |           +-- 停止重试（不再发射信号）
              +-- status == 200
                    +-- cancelRetry()（停止重试定时器，重置计数）
                    +-- parseJson(byteArray)
                          +-- 解析 yesterday -> mDay[0]
                          +-- 解析 forecast -> mDay[1..5]
                          +-- 解析 data -> mToday
                          +-- emit weatherDataReady(mToday, mDay)
                                +-- Widget::onWeatherDataReady()
                                      +-- updataUI()
                                      +-- QSettings 持久化 lastCityCode
                                      +-- DataCache::save()
                                      +-- ChartWidget::setData()
```

### 2.5 温度曲线绘制的触发时机

1. **数据返回后**：onWeatherDataReady 中调用 setData，触发 paintEvent。
2. **窗口重绘事件**：窗口被遮挡后重新显示时触发 paintEvent。

### 3.3 ApiClient 类（apiclient.h / ApiClient.cpp）

**职责：**

- 发送 HTTP GET 请求获取天气数据
- 通过 IP 定位获取当前城市
- 解析返回的 JSON 数据，填充 Today 和 Day 结构体
- 通过信号将数据传递给 Widget
- 失败时后台自动重试（最多 10 次，间隔 30 秒）

**关键成员变量：**

| 变量 | 类型 | 说明 |
|------|------|------|
| mNetAccessManger | QNetworkAccessManager* | 网络管理对象 |
| m_retryTimer | QTimer* | 重试定时器（单次触发模式） |
| mToday | Today | 今日天气数据（解析结果） |
| mDay[6] | Day[6] | 6 天天气数据 |
| m_retryCount | int | 当前请求已重试次数 |
| m_pendingCityCode | QString | 正在请求的城市编码（重试时复用） |

```cpp
static QString getCityCode(QString cityName){
    if(mCityMap.isEmpty()){
        initCityMap();
    }
    QMap<QString,QString>::iterator it = mCityMap.find(cityName);
    if(it == mCityMap.end()){
        it = mCityMap.find(cityName+"市");
    }
    if(it == mCityMap.end() && cityName.endsWith("市")){
        it = mCityMap.find(cityName.left(cityName.size() - 1));
    }
    if(it == mCityMap.end()){
        it = mCityMap.find(cityName+"县");
    }
    if(it == mCityMap.end() && cityName.endsWith("县")){
        it = mCityMap.find(cityName.left(cityName.size() - 1));
    }
    if(it!=mCityMap.end()){
        return it.value();
    }
    return "";
}
```

## 5. 核心代码段解析

### 5.1 getLocationByIP + 拼音查城市编码

```cpp
void ApiClient::getLocationByIP()
{
    auto *ipManager = new QNetworkAccessManager(this);
    connect(ipManager, &QNetworkAccessManager::finished,
            this, [=](QNetworkReply *reply) {
        reply->deleteLater();
        ipManager->deleteLater();
        if (reply->error() != QNetworkReply::NoError) {
            getWeatherInfo("北京");
            return;
        }
        QJsonDocument doc =
            QJsonDocument::fromJson(reply->readAll());
        QString cityPinyin =
            doc.object().value("city").toString();
        QString cityCode =
            WeatherTool::getCityCodeByPinyin(cityPinyin);
        getWeatherInfo(cityCode.isEmpty() ? "北京" : cityCode);
    });
    ipManager->get(QNetworkRequest(
        QUrl("https://ipinfo.io/json")));
}
```

**分析：**
- 使用独立 QNetworkAccessManager 避免信号冲突。
- lambda 中调用 deleteLater 确保资源释放。
- 定位失败时默认查询北京，保证程序在任何情况下都能显示天气数据。

### 5.2 getWeatherInfo 中城市名 vs 城市编码的判断

- 保存城市编码到 `m_pendingCityCode` 供重试时使用。

### 5.3 parseJson 解析过程

```cpp
void ApiClient::parseJson(const QByteArray &byteArray)
{
    QJsonParseError error;
    QJsonDocument doc = QJsonDocument::fromJson(byteArray, &error);
    if(error.error != QJsonParseError::NoError) return;
    QJsonObject rootObj = doc.object();
    if(rootObj.value("status").toInt() != 200){
        emit errorOccurred("请求数据失败，请检查城市编码或网络连接~~！");
        return;
    }
    // 1. 日期和城市
    mToday.date = rootObj.value("date").toString();
    mToday.city = rootObj.value("cityInfo")
                    .toObject().value("city").toString();
    // 2. 解析 yesterday -> mDay[0]
    QJsonObject objData = rootObj.value("data").toObject();
    QJsonObject ojbYesterday = objData.value("yesterday").toObject();
    // ... 提取 week, ymd, type, high, low, fx, fl, aqi
    // 3. 解析 forecast[0..4] -> mDay[1..5]
    QJsonArray forecastArray = objData.value("forecast").toArray();
    for(int i = 0; i < 5; i++){
        QJsonObject objForecast = forecastArray[i].toObject();
        // ... 提取各字段
    }
    // 4. 解析今天详情数据
    mToday.ganmao = objData.value("ganmao").toString();
    mToday.wendu = objData.value("wendu").toString().toDouble();
    mToday.shidu = objData.value("shidu").toString();
    mToday.pm25 = objData.value("pm25").toDouble();
    mToday.quality = objData.value("quality").toString();
    // 5. 从 forecast[0] 同步到今天
    mToday.type = mDay[1].type;
    mToday.fx = mDay[1].fx;
    mToday.fl = mDay[1].fl;
    mToday.high = mDay[1].high;
    mToday.low = mDay[1].low;
    // 6. 发射信号
    emit weatherDataReady(mToday, mDay);
}
```

**关键解析细节：**
- 温度字符串 "最高温度 26℃" 通过 split(" ") 分割后取第二部分，再用 left 去掉 "℃"。
- yesterday 数据映射到 mDay[0]，forecast 映射到 mDay[1..5]，data 顶层补充 mToday 剩余字段。
- mToday 的 type/fx/fl/high/low 来自 forecast[0]（即 mDay[1]），因为 API 在 forecast 中提供了今天的天气类型。

### 5.4 网络错误自动重试（后台 30 秒定时器）

当前实现（ApiClient.cpp）：

```cpp
void ApiClient::onReplied(QNetworkReply *reply)
{
    int status_code = reply->attribute(QNetworkRequest::HttpStatusCodeAttribute).toInt();

    if(reply->error() != QNetworkReply::NoError || status_code != 200)
    {
        if (m_retryCount == 0) {
            emit errorOccurred("请求数据失败，请检查城市编码或网络连接~~！");
        }
        if (++m_retryCount <= MAX_RETRY_COUNT) {
            qDebug() << "后台重试" << m_retryCount << "/" << MAX_RETRY_COUNT;
            m_retryTimer->start(RETRY_INTERVAL_MS);
        }
    }
    else
    {
        cancelRetry();
        QByteArray byteArray = reply->readAll();
        parseJson(byteArray);
    }
    reply->deleteLater();
}

void ApiClient::cancelRetry()
{
    m_retryTimer->stop();
    m_retryCount = 0;
}

void ApiClient::retry()
{
    QUrl url("http://t.weather.itboy.net/api/weather/city/" + m_pendingCityCode);
    mNetAccessManger->get(QNetworkRequest(url));
}
```

**重试机制分析：**

| 特性 | 当前值 |
|------|--------|
| 最大重试次数 | 10（MAX_RETRY_COUNT） |
| 重试间隔 | 30 秒（RETRY_INTERVAL_MS） |
| 定时器类型 | QTimer（单次触发，m_retryTimer->setSingleShot(true)） |
| 首次失败行为 | 立即 emit errorOccurred（弹窗提示用户） |
| 后续重试行为 | 后台静默，不弹窗，仅输出调试日志 |
| 成功时行为 | cancelRetry() 停止定时器，重置计数 |
| 新请求时行为 | getWeatherInfo 开头调用 cancelRetry() |
| 重试耗尽行为 | 停止重试，不再发射信号 |

**设计要点：**
- 用户只在第一次失败时收到弹窗提示，后续重试在后台静默进行，不打扰用户。
- 30 秒间隔避免频繁请求压垮服务器，同时给网络恢复留出时间。
- 重试时复用 `m_pendingCityCode`（在 `getWeatherInfo` 中保存），无需重新查表。
- 自动定位（`getLocationByIP`）的失败默认降级到北京。

### 5.6 drawCurve 温度曲线绘制

```cpp
void ChartWidget::paintEvent(QPaintEvent *event)
{
    Q_UNUSED(event);
    QPainter painter(this);
    painter.setRenderHint(QPainter::Antialiasing, true);

    // X 坐标：均匀分布
    int pointX[6];
    int stepX = width() / 7;
    for (int i = 0; i < 6; ++i)
        pointX[i] = stepX * (i + 1);

    // Y 坐标：基于均值偏移
    int tempSum = 0;
    for (int i = 0; i < 6; ++i) tempSum += m_temps[i];
    int tempAverage = tempSum / 6;
    int yCenter = height() / 2;
    for (int i = 0; i < 6; ++i)
        pointY[i] = yCenter - ((m_temps[i] - tempAverage) * 3);

    // 画圆点和温度标签
    QPen pen = painter.pen();
    pen.setWidth(1);
    pen.setColor(m_color);
    painter.setPen(pen);
    painter.setBrush(m_color);
    for (int i = 0; i < 6; ++i) {
        painter.drawEllipse(QPoint(pointX[i], pointY[i]), 3, 3);
        painter.drawText(pointX[i]-12, pointY[i]-12,
            QString::number(m_temps[i]) + "°");
    }

    // 画折线：第1段虚线，其余实线
    for (int i = 0; i < 5; ++i) {
        if (i == 0)
            pen.setStyle(Qt::DotLine);
        else
            pen.setStyle(Qt::SolidLine);
        painter.setPen(pen);
        painter.drawLine(pointX[i], pointY[i],
                       pointX[i+1], pointY[i+1]);
    }
}
```

**绘制原理：**
- X 坐标：width()/7 作为步长，6 个点均匀分布，左右留白。
- Y 坐标：以平均温度对应的 yCenter 为基准，高于均值偏上，低于均值偏下，差值乘以 3 放大视觉差异。
- 虚线 vs 实线：i==0（昨天到今天）用虚线表示历史数据；i>=1 用实线表示预报数据。
- 圆点半径 3px，温度标签偏移 (-12, -12)。

```cpp
bool Widget::eventFilter(QObject *obj, QEvent *event)
{
    if (event->type() == QEvent::MouseButtonPress && m_cityDropdown->isVisible()) {
        QWidget *src = qobject_cast<QWidget*>(obj);
        if (src && src != m_cityDropdown && !m_cityDropdown->isAncestorOf(src) && src != ui->lineEdit_2) {
            m_cityDropdown->hide();
        }
    }
    if (obj == ui->lineEdit_2 && event->type() == QEvent::MouseButtonPress) {
        if (m_cityDropdown->isVisible())
            m_cityDropdown->hide();
        else
            showCityDropdown();
        return false;
    }
    return QWidget::eventFilter(obj, event);
}
```

**设计要点：**

## 6. 架构总结

### 6.1 架构模式

本项目采用了 **分层架构 + 信号槽通信** 的模式，可视为轻量级 MVC 变体：

| 层级 | 对应组件 | 职责 |
|------|----------|------|
| View（视图） | widget.ui + Widget 类 | 界面布局、用户交互、数据展示 |
| Model（模型） | weatherdata.h（Today/Day） | 数据结构和业务实体 |
| Controller（控制器） | Widget + ApiClient + DataCache | 调度网络请求、本地缓存、控制数据流向 |
| Service（服务层） | ApiClient + WeatherTool | 网络通信、JSON 解析、编码查表 |
| Cache（缓存层） | DataCache | SQLite 存储，LRU 淘汰，可枚举缓存城市 |
| Utility（工具层） | WeatherTool | 静态工具方法 |
| Custom Widget | ChartWidget | 温度曲线绘制组件 |

通信机制：Qt 信号与槽
- ApiClient 发射 weatherDataReady 信号
- Widget 通过 connect 接收信号更新视图
- 用户操作通过 Ui 自动连接触发槽函数

### 6.2 优点

1. **关注点分离清晰**：网络请求封装在 ApiClient，缓存封装在 DataCache，温度曲线绘制独立为 ChartWidget。
2. **数据模型简洁**：Today/Day 结构体定义明确，默认构造函数提供兜底值增强健壮性。
3. **信号槽解耦**：ApiClient 通过信号发射数据，不依赖具体实现，易于扩展。
4. **SQLite 本地缓存 + LRU 淘汰**：减少 HTTP 请求频率，断网时可加载本地快照。最多保留 5 个城市，自动淘汰最早使用的记录。搜索框点击弹出最近缓存的城市列表，便于快速切换。
5. **自动定位 + 持久化**：IP 定位 + QSettings，首次自动定位，后续使用上次城市。
6. **窗口位置记忆**：QSettings 保存窗口位置，QTimer::singleShot 延迟恢复，屏幕可见性校验防止飞出。
7. **系统托盘支持**：右键菜单最小化到托盘，托盘图标常驻，单击/双击恢复窗口。
8. **网络错误自动重试**：HTTP 请求失败后首次立即弹窗提示，后台每 30 秒静默重试最多 10 次。
9. **资源管理规范**：图标和编码表通过 Qt 资源系统嵌入，便于发布。
10. **UI 细节完善**：无边框拖拽、右键退出、AQI 颜色分级、曲线虚实线区分、菜单黑色字体、城市下拉列表、Enter 键搜索。

### 6.3 缺点与改进空间

1. **项目组织散乱**：所有文件在根目录平铺，未按功能分目录（计划 CMake 迁移时一并处理）。
2. **曲线精度不够**：INCREMENT=3 硬编码，大温差时可能越界，需做 clamp 处理。（注：6 天预报温差通常 ≤20°C，当前参数安全）
3. **缺少国际化**：UI 字符串硬编码为中文，API 仅支持中国城市。
4. **CMake 未迁移**：当前使用 qmake 构建，计划新建项目时用 CMake 重建。
