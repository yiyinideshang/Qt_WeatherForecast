**步骤 5：加载 iconfont 图标字体**

```cpp
loadIconFont();

//加载 iconfont.ttf + 解析 iconfont.json，构建 mIconCodeMap
void Widget::loadIconFont()
{
    // 1.加载 .ttf 字体文件
    int fontId = QFontDatabase::addApplicationFont(":/res/weatherforecast_icon/iconfont.ttf");
    if (fontId >= 0) {
        QStringList families = QFontDatabase::applicationFontFamilies(fontId);
        if (!families.isEmpty())
            //QFont m_iconFont;
            m_iconFont = QFont(families.first(), 40);
    }

    // 2. 打开并检查JSON文件
    QFile f(":/res/weatherforecast_icon/iconfont.json");
    if (!f.open(QIODevice::ReadOnly))//以只读模式尝试打开这个文件。
        return;//打开失败立即退出
    QJsonDocument doc = QJsonDocument::fromJson(f.readAll());//读取并解析成JSON数据
    f.close();//关闭JSON文件
    if (!doc.isObject())//根据iconfont.json文件可知,它的最外层是一个{}JSON对象,如果不是可能是因为:文件是空的、被篡改了
        return;//返回JSON对象失败立即返回

    // 3. 解析 .json 文件，建立映射表
    QJsonArray glyphs = doc.object().value("glyphs").toArray();
    for (const QJsonValue &v : glyphs) {
        QJsonObject g = v.toObject();
        QString name = g.value("name").toString();      // 比如取到 "dayu"
        QString unicodeHex = g.value("unicode").toString(); // 比如取到 "e678"
        //注意，这里的 unicode 存的是十六进制字符串（不带 0x 前缀），比如 "e678"，而不是真正的数字。

        bool ok;//ok 是一个布尔引用，用来告诉你转换是否成功
        uint code = unicodeHex.toUInt(&ok, 16);//把字符串 "e678" 当成 16 进制数，转换成无符号整数 0xE695

        // QMap<QString, QChar> mIconCodeMap;
        if (ok && !name.isEmpty())//只插入合法数据：只有当转换成功（ok == true）并且名字不为空时，才插入到 QMap 中。
            mIconCodeMap.insert(name, QChar(code));//把整数 e678 强制转成 Qt 的字符类型 QChar
        // 此时的 mIconCodeMap长这样：
        // Key("dayu")    -> Value(QChar(0xE678))
        // Key("duoyun")  -> Value(QChar(0xE67f))
    }

    // 4. 人工处理别名
    struct { const char *api; const char *iconfont; } aliases[] = {
        {"狂风", "狂爆风"},
        {"强风/劲风", "强风劲风"},
        {"轻风", "清风"},
        {"雨夹雪", "雨加雪"},
    };
    for (auto &a : aliases) {
        auto it = mIconCodeMap.find(QLatin1String(a.iconfont));
        if (it != mIconCodeMap.end())
            mIconCodeMap.insert(QLatin1String(a.api), it.value());
    }
}

```

- `loadIconFont()` 加载 `iconfont.ttf` 到字体数据库，解析 `iconfont.json` 中 94个 glyph 名称到 Unicode 码点的映射，存入 `mIconCodeMap<QString, QChar>`。
- 别名映射将 API 返回的非标准天气名称（如"狂风"）映射到 iconfont 中实际的 glyph 名称（如"狂爆风"）。
- 彻底替代了旧的 `mTypeMap`（约 50 个 PNG 位图映射），用单个 ttf 字体文件+JSON 驱动，任意尺寸无损渲染。

**步骤 9：立即显示 UI + 后台网络请求**

```cpp
    QSettings settings;
    QString lastCityCode = settings.value("lastCityCode").toString();

    if ((!lastCityCode.isEmpty() && m_cache->load(lastCityCode, mToday, mDay)) ||
        loadRecentCache(lastCityCode)) {
        updataUI();
        m_currentCityCode = lastCityCode;
        m_startupCacheLoaded = true;
    } else
        showDefaultUI();

    m_lastRequestedCityCode = lastCityCode;  // 确保 onWeatherDataReady 能回退到正确 code

    QTimer::singleShot(0, this, [this, lastCityCode]() {
        if (!lastCityCode.isEmpty())
            m_apiClient->getWeatherInfo(lastCityCode);
        else
            m_apiClient->getLocationByIP();
    });
```

- **零等待原则**：先调用 UI 函数立即渲染（缓存数据或占位符），用户感知不到加载延迟。
- **缓存命中** → `m_cache->load` 填入真实数据，`updataUI()` 展示。
- **缓存降级**：若 `lastCityCode` 的缓存未命中，调用 `loadRecentCache()` 尝试加载最近访问的城市（从 `getAllCachedCities()` 取第一条），提高离线可用性。
- **无缓存** → `showDefaultUI()` 直接设置占位符（"--℃"、"加载中..."、"--"），不依赖 `weatherdata.h` 的默认构造值。
- `m_currentCityCode` 记录当前加载的城市编码，用于后续搜索时去重。
- `m_startupCacheLoaded` 标志控制首次失败时的弹窗文案（"网络连接失败，当前显示缓存数据" vs "请求数据失败"）。
- **QTimer::singleShot(0)** 将网络请求推迟到事件循环启动后执行，避免 `m_highChart`/`m_lowChart` 尚未初始化就被访问。
- 四级回退：缓存命中 → 显示缓存 + 后台刷新；缓存降级 → 显示最近城市缓存 + 后台请求原始城市；有历史无缓存 → 显示占位符 + 请求历史城市；首次 → 显示占位符 + IP 定位。

- 托盘图标常驻，窗口隐藏后仍显示。
- 使用 showNormal() 而非 show()，确保任务栏最小化的窗口也能正确恢复。
- raise() + activateWindow() 强制窗口置顶，避免藏在其他窗口后面。

### 2.3 缓存 + 网络请求决策流程

Widget 构造函数中，四级回退判断：

```
Widget 构造函数
  +-- QSettings 读取 lastCityCode
  |     +-- 为空（首次运行）
  |     |     +-- showDefaultUI() → 占位符
  |     |     +-- ApiClient::getLocationByIP()
  |     |           +-- GET http://ip-api.com/json/?lang=zh-CN
  |     |           +-- 解析 city 中文名
  |     |           +-- 若 city 未找到则用 regionName 降级
  |     |           +-- getCityCode() 查编码
  |     |           +-- getWeatherInfo(cityCode)
  |     +-- 非空
  |           +-- DataCache::load(lastCityCode)
  |           |     +-- 命中且未过期 → updataUI()
  |           |     |     +-- m_currentCityCode = lastCityCode
  |           |     |     +-- m_startupCacheLoaded = true
  |           |     |     +-- QTimer::singleShot → 后台网络刷新
  |           |     +-- 未命中或过期
  |           |           +-- loadRecentCache(lastCityCode)
  |           |           |     +-- 有缓存记录 → updataUI()（显示最近城市）
  |           |           |     |     +-- m_startupCacheLoaded = true
  |           |           |     |     +-- QTimer::singleShot → 请求原始 lastCityCode
  |           |           |     +-- 无缓存记录 → showDefaultUI()
  |           |           |           +-- QTimer::singleShot → 请求原始 lastCityCode
  +-- 等待数据（来自缓存或网络）
```

### 2.4 数据返回后的信号链与 UI 更新

```
服务器返回数据
  +-- QNetworkAccessManager::finished(QNetworkReply*)
        +-- ApiClient::onReplied()
              +-- status != 200 或 reply->error()
              |     +-- m_retryCount == 0 → emit errorOccurred（弹窗提示用户）
              |     +-- ++m_retryCount <= MAX_RETRY_COUNT (10)
              |     |     +-- m_retryTimer->start(30000ms) → retry()
              |     |           +-- 重新 GET 请求（后台静默重试）
               |     +-- m_retryCount > MAX_RETRY_COUNT
               |           +-- emit errorOccurred("重连网络失败！")
              +-- status == 200
                    +-- cancelRetry()（停止重试定时器，重置计数）
                    +-- parseJson(byteArray)
                          +-- 确定今天基准日期（data.time 字段）
                          +-- 在 forecast 中匹配 todayIdx
                          +-- 解析 yesterday/fc[todayIdx-1] -> mDay[0]
                          +-- 解析 fc[todayIdx..todayIdx+5] -> mDay[1..6]
                          +-- 解析 pm10/sunrise/sunset/notice/updateTime
                          +-- emit weatherDataReady(mToday, mDay)
                                +-- Widget::onWeatherDataReady()
                                      +-- updataUI()
                                      +-- QSettings 持久化 lastCityCode
                                      +-- DataCache::save()
                                      +-- ChartWidget::setData()
```

### 2.5 温度曲线绘制的触发时机

1. **数据返回后**：onWeatherDataReady 中调用 setData，触发 paintEvent。
2. **窗口重绘事件**：窗口被遮挡后重新显示时触发 paintEvent。

```cpp
    // 3. 未匹配且是重名城市 → 返回 ""（歧义，由调用者弹窗提示）
```

## 4. 数据流分析

### 4.1 HTTP 请求 URL


### 4.3 parseJson 解析过程

```
rootObj
  +-- "cityInfo"."city" -> mToday.city
  +-- "data"
        +-- "time" -> 确定今天基准日期（三处兜底）
        |     +-- data.time "yyyy-MM-dd HH:mm:ss"
        |     +-- rootObj.time
        |     +-- QDate::currentDate()
        +-- 在 forecast[] 中匹配 todayIdx (ymd == todayDate)
        +-- 昨天
        |     +-- todayIdx==0 → data.yesterday -> mDay[0]
        |     +-- todayIdx>0  → forecast[todayIdx-1] -> mDay[0]
        +-- 今天 + 未来五天
        |     +-- forecast[todayIdx..todayIdx+5] -> mDay[1..6]
        |     +-- 每个 day 解析: ymd, week, type, sunrise, sunset,
        |         notice, high, low, fx, fl, aqi
        +-- 今日详情
        |     +-- ganmao, wendu(toDouble), shidu, pm25(toDouble),
        |         pm10(toDouble), quality, updateTime(三处兜底)
        +-- 同步 forecast 到 mToday
              +-- mToday.type/fx/fl/high/low = mDay[1]
```

**关键设计——预报日期对齐：**

凌晨 00:00~06:00 时段，API 的 `data.forecast[0]` 可能仍为昨天的数据，而 `rootObj.date` 也可能滞后一天。因此：
1. **基准时间来自 `data.time`**（服务器真实系统时间），而非 `rootObj.date`。
2. 在 `forecast[]` 中按 `ymd` 匹配 `todayDate`，找到 `todayIdx`。
3. `mDay[0]` 来自 `forecast[todayIdx-1]`（若 `todayIdx==0` 则用 `yesterday` 字段）。
4. `mDay[1..6]` 来自 `forecast[todayIdx..todayIdx+5]`。
5. 顶部 `lblDate` 改用 `QDate::currentDate()` + `QLocale("zh_CN").dayName()`，永远显示正确日历日期。

```cpp
mToday.type = mDay[1].type;

mToday.high = mDay[1].high;
mToday.low  = mDay[1].low;
```

### 5.1 getLocationByIP + 中文城市名查编码（regionName 降级）

```cpp
void ApiClient::getLocationByIP()
{
    //1. 创建一个临时的网络管理器（QNetworkAccessManager）
    auto *ipManager = new QNetworkAccessManager(this);
    //3. 在收到回复的槽函数（Lambda 表达式）里：
    connect(ipManager, &QNetworkAccessManager::finished, this, [=](QNetworkReply *reply) {
        //① 延时释放网络资源
        reply->deleteLater();
        ipManager->deleteLater();
        //②检查回复是否有错误（如网络不通、超时、HTTP 状态码错误等）。
        if (reply->error() != QNetworkReply::NoError) {
            getWeatherInfo("北京");
            return;
        }
        // ③ 解析 JSON，提取城市名
        QJsonDocument doc = QJsonDocument::fromJson(reply->readAll());
        QString cityName  = doc.object().value("city").toString();
        QString cityCode = WeatherTool::getCityCode(cityName);//根据城市名查编码（用于 IP 定位）
        QString region = doc.object().value("regionName").toString();
        if (cityCode.isEmpty()) {
            cityCode = WeatherTool::getCityCode(region);//城市名不存在时,根据地区名查编码
        }
        //输出 地区名 + 城市名
        qDebug() << "自动定位到:" <<region << cityName  << "code:" << cityCode;

        //如果 cityCode 为空，默认使用“北京”，否则就用定位城市的编码请求天气。
        getWeatherInfo(cityCode.isEmpty() ? "北京" : cityCode);
    });
    //2. 发送一个 GET 请求到 http://ip-api.com/json/?lang=zh-CN
    ipManager->get(QNetworkRequest(QUrl("http://ip-api.com/json/?lang=zh-CN")));
}

```

**分析：**
- 使用独立 QNetworkAccessManager 避免信号冲突。
- lambda 中调用 deleteLater 确保资源释放。
- 定位失败时默认查询北京，保证程序在任何情况下都能显示天气数据。
- city 在本地城市库未命中时自动用 regionName（省份名）再次尝试匹配，提升定位覆盖率。

### 5.2 getWeatherInfo 中城市名 vs 城市编码的判断

```cpp
void ApiClient::getWeatherInfo(const QString &cityName)
{
    QString cityCode = cityName;

    if (cityName.startsWith("101") && cityName.length() == 9) {
        qDebug() << "直接使用城市编码:" << cityCode;
    } else {
        cityCode = WeatherTool::getCityCode(cityName);
        if(cityCode.isEmpty()){
            emit errorOccurred("请检查输入的城市是否正确!(省级以下的城市)");
            return;     // ← 城市不存在时直接 return，不取消旧重试
        }
    }

    cancelRetry();  // ← 确认城市有效后才取消旧重试
    m_pendingCityCode = cityCode;

    QUrl url("http://t.weather.itboy.net/api/weather/city/"
            + cityCode);
    mNetAccessManger->get(QNetworkRequest(url));
}
```

**分析：**
- 支持双重入参：城市名（"北京"）或城市编码（"101010100"）。
- 城市编码以 "101" 开头且长度为 9 位，通过此特征自动区分。
- `cancelRetry()` 从函数开头移到 cityCode 验证成功之后：城市不存在时不取消旧重试（修复 1D/2D 场景）。
- 保存城市编码到 `m_pendingCityCode` 供重试时使用。

### 5.3 parseJson 解析过程（带预报日期对齐）

```cpp
void ApiClient::parseJson(const QByteArray &byteArray)
{
    QJsonParseError error;
    QJsonDocument doc = QJsonDocument::fromJson(byteArray, &error);
    if(error.error != QJsonParseError::NoError) return;
    QJsonObject rootObj = doc.object();
    if(rootObj.value("status").toInt() != 200){
        emit errorOccurred("请求数据失败，请检查城市编码或网络连接~~！");
        return;
    }

    // 1. 解析城市
    mToday.city = rootObj.value("cityInfo")
                    .toObject().value("city").toString();

    // 2. 确定今天的基准日期（从 data.time 字段，三处兜底）
    QJsonObject objData = rootObj.value("data").toObject();
    QJsonArray forecastArray = objData.value("forecast").toArray();
    QString timeStr = objData.value("time").toString();
    if (timeStr.isEmpty())
        timeStr = rootObj.value("time").toString();
    QDate todayDate;
    QDateTime dt = QDateTime::fromString(timeStr, "yyyy-MM-dd HH:mm:ss");
    if (dt.isValid()) todayDate = dt.date();
    if (!todayDate.isValid()) {
        dt = QDateTime::fromString(timeStr, Qt::ISODate);
        if (dt.isValid()) todayDate = dt.date();
    }
    if (!todayDate.isValid())
        todayDate = QDate::currentDate();

    // 3. 在 forecast 中找到匹配今天的索引
    int todayIdx = -1;
    for (int i = 0; i < forecastArray.size(); i++) {
        QDate fd = QDate::fromString(
            forecastArray[i].toObject().value("ymd").toString(), "yyyy-MM-dd");
        if (fd == todayDate) { todayIdx = i; break; }
    }
    if (todayIdx < 0) todayIdx = 0;

    // 4. 按对齐后的索引填充 mDay[0..6]
    auto parseDay = [](const QJsonObject &obj, Day &d) {
        d.date = obj.value("ymd").toString();
        d.week = obj.value("week").toString();
        d.type = obj.value("type").toString();
        d.sunrise = obj.value("sunrise").toString();
        d.sunset = obj.value("sunset").toString();
        d.notice = obj.value("notice").toString();
        QString s;
        QStringList parts = obj.value("high").toString().split(" ");
        s = parts.size() > 1 ? parts.at(1) : "";
        s = s.left(s.length()-1);
        d.high = s.toInt();
        parts = obj.value("low").toString().split(" ");
        s = parts.size() > 1 ? parts.at(1) : "";
        s = s.left(s.length()-1);
        d.low = s.toInt();
        d.fl = obj.value("fl").toString();
        d.fx = obj.value("fx").toString();
        d.aqi = obj.value("aqi").toDouble();
    };

    // 昨天
    if (todayIdx == 0)
        parseDay(objData.value("yesterday").toObject(), mDay[0]);
    else
        parseDay(forecastArray[todayIdx - 1].toObject(), mDay[0]);

    // 今天 + 未来五天
    int endIdx = qMin(todayIdx + 6, forecastArray.size());
    for (int i = todayIdx; i < endIdx; i++)
        parseDay(forecastArray[i].toObject(), mDay[i - todayIdx + 1]);

    // 5. 解析今天详情（pm10, updateTime 三处兜底）
    mToday.ganmao = objData.value("ganmao").toString();
    mToday.wendu = objData.value("wendu").toString().toDouble();
    mToday.shidu = objData.value("shidu").toString();
    mToday.pm25 = objData.value("pm25").toDouble();
    mToday.pm10 = objData.value("pm10").toDouble();
    mToday.quality = objData.value("quality").toString();
    mToday.updateTime = objData.value("time").toString();
    if (mToday.updateTime.isEmpty())
        mToday.updateTime = rootObj.value("time").toString();
    if (mToday.updateTime.isEmpty())
        mToday.updateTime = rootObj.value("cityInfo")
                            .toObject().value("updateTime").toString();
    mToday.date = rootObj.value("date").toString();

    // 6. 从对齐后的 mDay[1] 同步到今天
    mToday.type = mDay[1].type;
    mToday.fx = mDay[1].fx;
    mToday.fl = mDay[1].fl;
    mToday.high = mDay[1].high;
    mToday.low = mDay[1].low;

    // 7. 发射信号
    emit weatherDataReady(mToday, mDay);
}
```

**关键解析细节：**
- **日期对齐**：使用 `data.time` 字段（服务器真实系统时间）确定今天，在 `forecast[]` 中按 `ymd` 匹配索引，解决凌晨数据滚动错位问题（API 的 `date` 根字段可能滞后一天）。
- 温度字符串 "最高温度 26℃" 通过 split(" ") 分割后取第二部分，再用 left 去掉 "℃"。
- `updateTime` 三级回退：`data.time` → `rootObj.time` → `cityInfo.updateTime`。
- 新增解析字段：`pm10`、`sunrise`、`sunset`、`notice`。

### 5.4 网络错误自动重试（后台 30 秒定时器）

当前实现（ApiClient.cpp）：

```cpp
void ApiClient::onReplied(QNetworkReply *reply)
{
    int status_code = reply->attribute(QNetworkRequest::HttpStatusCodeAttribute).toInt();

    if(reply->error() != QNetworkReply::NoError || status_code != 200)
    {
        if (m_retryCount == 0) {
            emit errorOccurred("请求数据失败，请检查网络连接~~！");
        }
        if (++m_retryCount <= MAX_RETRY_COUNT) {
            qDebug() << "后台重试" << m_retryCount << "/" << MAX_RETRY_COUNT;
            m_retryTimer->start(RETRY_INTERVAL_MS);
        } else {
            qDebug() << "重试次数已耗尽";
            emit errorOccurred("重连网络失败！");  // ← 第 10 次重试失败后通知用户
        }
    }
    else
    {
        cancelRetry();
        QByteArray byteArray = reply->readAll();
        parseJson(byteArray);
    }
    reply->deleteLater();
}

void ApiClient::cancelRetry()
{
    m_retryTimer->stop();
    m_retryCount = 0;
}

void ApiClient::retry()
{
    QUrl url("http://t.weather.itboy.net/api/weather/city/" + m_pendingCityCode);
    mNetAccessManger->get(QNetworkRequest(url));
}
```

**重试机制分析：**

| 特性 | 当前值 |
|------|--------|
| 最大重试次数 | 10（MAX_RETRY_COUNT） |
| 重试间隔 | 30 秒（RETRY_INTERVAL_MS） |
| 定时器类型 | QTimer（单次触发，m_retryTimer->setSingleShot(true)） |
| 首次失败行为 | 立即 emit errorOccurred（弹窗提示用户） |
| 后续重试行为 | 后台静默，不弹窗，仅输出调试日志 |
| 成功时行为 | cancelRetry() 停止定时器，重置计数 |
| 新请求时行为 | getWeatherInfo 验证 cityCode 后调用 cancelRetry()（不存在城市不取消旧重试） |
| 重试耗尽行为 | `emit errorOccurred("重连网络失败！")` |

**设计要点：**
- 用户只在第一次失败时收到弹窗提示，后续重试在后台静默进行，不打扰用户。
- 30 秒间隔避免频繁请求压垮服务器，同时给网络恢复留出时间。
- 重试时复用 `m_pendingCityCode`（在 `getWeatherInfo` 中保存），无需重新查表。
- 自动定位（`getLocationByIP`）的失败默认降级到北京。
- IP 定位优先用 `city` 字段，若未找到则用 `regionName`（省份名）二次尝试，提高定位城市覆盖率。

- **日期改用系统时钟**：`QDate::currentDate()` + `QLocale("zh_CN").dayName()`，永远显示正确日历日期，不依赖 API 数据。
- **iconfont 图标**：`renderWeatherIcon()` 替代旧的 `mTypeMap` 位图方案，任意尺寸无损渲染，QPixmap 结果缓存于 `m_iconCache`。
- **新增标签**：`lblNotice`（温馨提示）、`lblPM10`、`lblSunrise`、`lblSunset`、`lblUpdateTime`。
- **FX/FL 语义对齐**：`mFxList`（`lblFX0~6`，row 0）显示 `mDay[i].fx`（风向）；`mFlList`（`lblFl0~6`，row 1）显示 `mDay[i].fl`（风级）。
- AQI 六色显示：优(绿)/良(黄)/轻度(橙)/中度(红)/重度(紫红)/严重(深紫)。
- QPixmap 缓存：`m_iconCache` 首次渲染后缓存，后续直接复用避免重复 QPainter 开销。

### 5.6 drawCurve 温度曲线绘制

```cpp
void ChartWidget::paintEvent(QPaintEvent *event)
{
    Q_UNUSED(event);
    QPainter painter(this);
    painter.setRenderHint(QPainter::Antialiasing, true);

    // X 坐标：均匀分布
    int pointX[7];
    int stepX = width() / 8;
    for (int i = 0; i < 7; ++i)
        pointX[i] = stepX * (i + 1);

    // Y 坐标：基于均值偏移
    int tempSum = 0;
    for (int i = 0; i < 7; ++i) tempSum += m_temps[i];
    int tempAverage = tempSum / 7;
    int yCenter = height() / 2;
    for (int i = 0; i < 7; ++i)
        pointY[i] = yCenter - ((m_temps[i] - tempAverage) * 3);

    // 画圆点和温度标签
    QPen pen = painter.pen();
    pen.setWidth(1);
    pen.setColor(m_color);
    painter.setPen(pen);
    painter.setBrush(m_color);
    for (int i = 0; i < 7; ++i) {
        painter.drawEllipse(QPoint(pointX[i], pointY[i]), 3, 3);
        painter.drawText(pointX[i]-12, pointY[i]-12,
            QString::number(m_temps[i]) + "°");
    }

    // 画折线：第1段虚线，其余实线
    for (int i = 0; i < 6; ++i) {
        if (i == 0)
            pen.setStyle(Qt::DotLine);
        else
            pen.setStyle(Qt::SolidLine);
        painter.setPen(pen);
        painter.drawLine(pointX[i], pointY[i],
                       pointX[i+1], pointY[i+1]);
    }
}
```

**绘制原理：**
- X 坐标：width()/8 作为步长，7 个点均匀分布，左右留白。
- Y 坐标：以平均温度对应的 yCenter 为基准，高于均值偏上，低于均值偏下，差值乘以 3 放大视觉差异。
- 虚线 vs 实线：i==0（昨天到今天）用虚线表示历史数据；i>=1 用实线表示预报数据。
- 圆点半径 3px，温度标签偏移 (-12, -12)。

**eventFilter 实现：**

```cpp
bool Widget::eventFilter(QObject *obj, QEvent *event)
{
    if (event->type() == QEvent::MouseButtonPress && m_cityDropdown->isVisible()) {
        QWidget *src = qobject_cast<QWidget*>(obj);
        if (src && src != m_cityDropdown && !m_cityDropdown->isAncestorOf(src) && src != ui->lineEdit_2) {
            m_cityDropdown->hide();
        }
    }
    if (obj == ui->lineEdit_2 && event->type() == QEvent::MouseButtonPress) {
        if (m_cityDropdown->isVisible())
            m_cityDropdown->hide();
        else
            showCityDropdown();
        return false;
    }
    return QWidget::eventFilter(obj, event);
}
```

**设计要点：**
- 缓存为空时显示 "暂无缓存记录"（灰色文字，不可点击）。
- 有缓存时列出所有城市名，每个 item 的 `UserRole+1` 存储对应的 cityCode，最后一项 "清除缓存记录" 带 `__clear__` UserRole 标记。
- 点击 item 时，优先用 `UserRole+1` 中的 cityCode 直接查询（绕过 `getCityCode` 歧义判断），数据源选择逻辑：缓存→延迟网络→回退到文本搜索。
- `__clear__` 标记在 itemClicked 连接中识别，调用 `m_cache->clearAll()` 清空缓存。
- 下拉列表面向 `lineEdit_2` 下方弹出，宽度与搜索框一致。
- 应用级事件过滤器（`qApp->installEventFilter(this)`）拦截全局鼠标事件：点击下拉外部任意位置（除非是 `lineEdit_2` 本身）则关闭下拉。
- `lineEdit_2` 的 MouseButtonPress 事件被 eventFilter 单独处理：若下拉已显示则隐藏，否则显示（切换逻辑）。
- `lineEdit_2` 禁用右键菜单（`setContextMenuPolicy(Qt::NoContextMenu)`）。
- `returnPressed` 信号连接 `on_pushButton_2_clicked`，支持 Enter 键搜索。
- `changeEvent` 重写：窗口最小化时自动隐藏下拉框（`WindowStateChange` 事件中检查 `isMinimized()` + `m_cityDropdown->isVisible()`）。

### 5.8 Iconfont 图标系统

**设计目标：** 使用单个 iconfont 字体文件（`iconfont.ttf`）+ JSON 配置文件替代约 50 个 PNG 位图，实现任意尺寸下无损渲染的天气图标。

**加载流程（`loadIconFont`）：**

```cpp
void Widget::loadIconFont()
{
    // 1. 注册字体文件
    int fontId = QFontDatabase::addApplicationFont(
        ":/res/weatherforecast_icon/iconfont.ttf");
    if (fontId >= 0) {
        QStringList families = QFontDatabase::applicationFontFamilies(fontId);
        if (!families.isEmpty())
            m_iconFont = QFont(families.first(), 40);
    }

    // 2. 解析 JSON，构建 name → QChar 映射
    QFile f(":/res/weatherforecast_icon/iconfont.json");
    f.open(QIODevice::ReadOnly);
    QJsonDocument doc = QJsonDocument::fromJson(f.readAll());
    QJsonArray glyphs = doc.object().value("glyphs").toArray();
    for (const QJsonValue &v : glyphs) {
        QJsonObject g = v.toObject();
        QString name = g.value("name").toString();
        uint code = g.value("unicode").toString().toUInt(&ok, 16);
        if (ok && !name.isEmpty())
            mIconCodeMap.insert(name, QChar(code));
    }

    // 3. API 返回别名 → iconfont 名称映射
    // {"狂风"→"狂爆风", "强风/劲风"→"强风劲风", ...}
    for (auto &a : aliases) { ... }
}
```

**渲染流程（`renderWeatherIcon`）：**

```cpp
QPixmap Widget::renderWeatherIcon(const QString &weather)
{
    // 1. 检查缓存
    auto it = m_iconCache.constFind(weather);
    if (it != m_iconCache.constEnd())
        return it.value();

    // 2. 查映射表，获取 Unicode 码点
    QChar code(0xe695);  // 默认"晴"
    auto codeIt = mIconCodeMap.constFind(weather);
    if (codeIt != mIconCodeMap.constEnd())
        code = codeIt.value();

    // 3. QPainter 渲染到 QPixmap
    QPixmap pixmap(64, 64);
    pixmap.fill(Qt::transparent);
    QPainter painter(&pixmap);
    painter.setRenderHint(QPainter::Antialiasing);
    QFont f(m_iconFont);
    f.setPixelSize(48);
    painter.setFont(f);
    painter.setPen(QColor(255, 255, 255));
    painter.drawText(QRect(0, 0, 64, 64), Qt::AlignCenter, QString(code));
    painter.end();

    // 4. 缓存并返回
    m_iconCache.insert(weather, pixmap);
    return pixmap;
}
```

**设计要点：**
- `iconfont.json` 包含 80+ glyph，每个 glyph 有 `name`（如"晴"、"多云"）和 `unicode`（如"e695"）两个字段。
- 别名映射：API 返回的某些天气名称与 iconfont 的 glyph 名称不一致（如 API "狂风" vs iconfont "狂爆风"），在 `loadIconFont` 末尾补充映射。
- `m_iconCache` 缓存 `QString → QPixmap`：首次调用 `renderWeatherIcon("晴")` 时渲染并缓存，后续相同 weather 字符串直接返回缓存的 QPixmap。
- 默认使用 `0xe695`（晴）作为兜底图标，在 `showDefaultUI` 和未知天气类型时生效。

**读取（Widget 构造函数中）：**

- `m_currentCityCode` 记录当前显示的城市编码。
- `m_startupCacheLoaded` 标记启动时已加载缓存，用于错误弹窗的差异化文案。

**写入（onWeatherDataReady 中）：**

### 5.10 on_pushButton_2_clicked 搜索逻辑

```cpp
void Widget::on_pushButton_2_clicked()
{
    QString cityName = ui->lineEdit_2->text();
    QString cityCode = WeatherTool::getCityCode(cityName);

    if (!cityCode.isEmpty() && cityCode == m_currentCityCode)
        return;  // 重复搜索同一城市，跳过

    m_startupCacheLoaded = false;
    m_apiClient->cancelRetry();

    if (!cityCode.isEmpty()) {
        m_lastRequestedCityCode = cityCode;
        QSettings settings;
        settings.setValue("lastCityCode", cityCode);
        if (m_cache->load(cityCode, mToday, mDay)) {
            onWeatherDataReady(mToday, mDay);
            return;
        }
        m_apiClient->getWeatherInfo(cityCode);
        return;
    }

    // 编码为空 → 重名城市歧义（如"朝阳"、"通州"等）
    QStringList options = WeatherTool::getAmbiguousOptions(cityName);
    if (!options.isEmpty()) {
        QMessageBox msgBox(this);
        msgBox.setWindowTitle("提示");
        msgBox.setText("找到多个同名城市，请准确输入（如" + cityName + "(辽宁)）：");
        msgBox.setDetailedText(options.join("\n"));
        msgBox.exec();
    } else {
        m_lastRequestedCityCode.clear();
        m_apiClient->getWeatherInfo(cityName);
    }
}
```

**分析：**
- **去重**：如果当前 UI 中有城市（`cityCode` 非空）且搜索的城市编码与当前显示的城市相同，直接返回避免重复刷新。加上 `!cityCode.isEmpty()` 防止空编码（城市不存在）误匹配空 `m_currentCityCode`（无缓存启动时）。
- **清除重试**：仅在 `cityCode` 非空（城市有效）时调用 `cancelRetry()`；城市不存在时不取消旧重试，保留原 pending 城市的定时器继续运行。
- **重置状态**：`m_startupCacheLoaded = false`，确保搜索失败时弹出完整错误提示而非"后台重连"。
- **歧义弹窗**：`getCityCode` 返回空时调用 `getAmbiguousOptions` 判断是否重名城市；若是则弹窗列出所有 `城名(省)` 组合提示用户准确输入。
- 先查城市编码，若编码存在则优先尝试读缓存（可离线显示）。
- 即使缓存命中，也会设置 `lastCityCode` 以便下次启动恢复。
- 缓存未命中或无编码则发起网络请求。
- `m_lastRequestedCityCode` 在发起请求前记录，供 `onWeatherDataReady` 在结果城市重名时回退确认。

### 5.11 onWeatherDataReady 数据落地

```cpp
void Widget::onWeatherDataReady(const Today &today, const Day day[7])
{
    mToday = today;
    for (int i = 0; i < 7; ++i) mDay[i] = day[i];
    updataUI();

    QSettings settings;
    QString cityCode = WeatherTool::getCityCode(mToday.city);
    if (cityCode.isEmpty())
        cityCode = m_lastRequestedCityCode;  // 回退：结果城市重名时用请求 code

    if (!cityCode.isEmpty()) {
        settings.setValue("lastCityCode", cityCode);
        m_currentCityCode = cityCode;
    }

    int high[7], low[7];
    for (int i = 0; i < 7; ++i) {
        high[i] = mDay[i].high;
        low[i] = mDay[i].low;
    }
    m_highChart->setData(high);
    m_lowChart->setData(low);

    if (!cityCode.isEmpty())
        m_cache->save(cityCode, mToday, mDay);
}
```

**分析：**
- **歧义回退**：`getCityCode(mToday.city)` 可能因重名返回空，此时用 `m_lastRequestedCityCode` 作为回退编码。
- **save 守卫**：`cityCode.isEmpty()` 时跳过 m_cache->save，避免写入空编码或脏数据。
- 存 QSettings 和写 SQLite 都在数据到达后立即执行，确保下次启动可用。
- `m_currentCityCode` 同步更新，用于后续搜索去重。
- 曲线控件在 UI 更新后刷新，保证数据一致性。
- `m_cache->save()` 内部会调用 `evictIfNeeded()` 检查并执行 LRU 淘汰。

## 6. 架构总结

### 6.1 架构模式

本项目采用了 **分层架构 + 信号槽通信** 的模式，可视为轻量级 MVC 变体：

| 层级 | 对应组件 | 职责 |
|------|----------|------|
| View（视图） | widget.ui + Widget 类 | 界面布局、用户交互、数据展示 |
| Model（模型） | weatherdata.h（Today/Day） | 数据结构和业务实体 |
| Controller（控制器） | Widget + ApiClient + DataCache | 调度网络请求、本地缓存、控制数据流向 |
| Service（服务层） | ApiClient + WeatherTool | 网络通信、JSON 解析、编码查表 |
| Cache（缓存层） | DataCache | SQLite 存储，LRU 淘汰，可枚举缓存城市 |
| Utility（工具层） | WeatherTool | 静态工具方法 |
| Custom Widget | ChartWidget | 温度曲线绘制组件 |

通信机制：Qt 信号与槽
- ApiClient 发射 weatherDataReady 信号
- Widget 通过 connect 接收信号更新视图
- 用户操作通过 Ui 自动连接触发槽函数

### 6.2 优点

1. **关注点分离清晰**：网络请求封装在 ApiClient，缓存封装在 DataCache，温度曲线绘制独立为 ChartWidget。
2. **数据模型简洁**：Today/Day 结构体定义明确，默认构造函数提供兜底值增强健壮性。
3. **信号槽解耦**：ApiClient 通过信号发射数据，不依赖具体实现，易于扩展。
4. **SQLite 本地缓存 + LRU 淘汰**：减少 HTTP 请求频率，断网时可加载本地快照。最多保留 5 个城市，自动淘汰最早使用的记录。搜索框点击弹出最近缓存的城市列表，便于快速切换。
5. **自动定位 + 持久化**：IP 定位 + QSettings，首次自动定位，后续使用上次城市。
6. **窗口位置记忆**：QSettings 保存窗口位置，QTimer::singleShot(50ms) 延迟恢复，屏幕可见性校验防止飞出。
7. **系统托盘支持**：右键菜单最小化到托盘，托盘图标常驻，单击/双击恢复窗口。
8. **网络错误自动重试**：HTTP 请求失败后首次立即弹窗提示，后台每 30 秒静默重试最多 10 次。
9. **资源管理规范**：icontfont 字体 + JSON 配置 + 编码表通过 Qt 资源系统嵌入，仅 1 个 ttf + 1 个 json 替代约 50 个 PNG 位图。
10. **UI 细节完善**：无边框拖拽、右键退出、AQI 颜色分级 + PM10 + 日出日落 + 温馨提示、曲线虚实线区分、菜单黑色字体、城市下拉列表（最近缓存 + 清除缓存按钮）、Enter 键搜索、多标签占位符、重复搜索去重。
11. **预报日期对齐**：基于 `data.time` 字段匹配 forecast 索引，解决凌晨数据滚动错位，顶部日期使用系统时钟保证正确性。
12. **lblFX/lblFl 语义对齐**：标签名与数据字段含义一致（FX→风向，Fl→风级），消除命名歧义。
13. **缓存降级**：`lastCityCode` 缓存未命中时自动加载最近访问的城市缓存，提高离线可用性。
14. **城市歧义处理**：`WeatherTool` 支持 composite key (`城名(省)`)，重名城市弹窗提示 + 下拉列表绕过歧义，整个流程对用户透明。
15. **动态 font size**：`lblCity` 文本自适应（城市名过长时缩小字号），避免 UI 溢出。

### 6.3 缺点与改进空间

1. **项目组织散乱**：所有文件在根目录平铺，未按功能分目录（计划 CMake 迁移时一并处理）。
2. **曲线精度不够**：INCREMENT=3 硬编码，大温差时可能越界，需做 clamp 处理。（注：7 天预报温差通常 ≤20°C，当前参数安全）
3. **缺少国际化**：UI 字符串硬编码为中文，API 仅支持中国城市。
